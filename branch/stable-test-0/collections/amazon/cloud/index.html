Plugin Index

These are the plugins in the amazon.cloud collection:

Modules

  • autoscaling_launch_configuration module – Creates and manages a web ACL association.

  • autoscaling_lifecycle_hook module – Creates and manages an Auto Scaling group

  • autoscaling_warm_pool module – Creates and manages a pool of pre-initialized EC2 instances that sits alongside the Auto Scaling group

  • backup_backup_vault module – Create and manage logical containers where backups are stored

  • backup_framework module – Create and manage frameworks with one or more controls

  • backup_report_plan module – Create and manage report plans

  • cloudtrail_event_data_store module – Creates and manages a new event data store

  • cloudtrail_trail module – Creates and manages a trail that specifies the settings for delivery of log data to an Amazon S3 bucket.

  • cloudwatch_composite_alarm module – Creates and manages a composite alarm

  • cloudwatch_metric_stream module – Creates and manages a metric stream

  • dynamodb_global_table module – Creates and manages a Version 2019.11.21 global table

  • ec2_placement_group module – Creates and manages a placement group

  • ecr_repository module – Creates and manages an AWS ECR Repository resource

  • ecs_capacity_provider module – Creates and manages a capacity provider

  • ecs_cluster module – Creates and manages an Amazon Elastic Container Service (Amazon ECS) cluster

  • ecs_cluster_capacity_provider_associations module – Creates and manages the association between one or more capacity providers and a default capacity provider strategy with a cluster

  • ecs_primary_task_set module – Specifies which task set in a service is the primary task set.

  • eks_addon module – Creates and manages Amazon EKS add-ons

  • eks_cluster module – Create and manages Amazon EKS control planes

  • eks_fargate_profile module – Creates and manage AWS Fargate profiles

  • iam_instance_profile module – Creates and manages an instance profile

  • iam_role module – Create and manage roles

  • iam_server_certificate module – Uploads and manages a server certificate entity for the AWS account

  • kms_alias module – Specifies a display name for a KMS key.

  • kms_replica_key module – Creates and manages a multi-Region replica key that is based on a multi-Region primary key

  • lambda_code_signing_config module – Code signing for AWS Lambda

  • lambda_event_source_mapping module – Create a mapping between an event source and an AWS Lambda function

  • lambda_function module – Create and manage Lambda functions

  • logs_log_group module – Create and manage log groups

  • logs_metric_filter module – Specifies a metric filter that describes how CloudWatch Logs extracts information from logs and transforms it into Amazon CloudWatch metrics

  • logs_query_definition module – Create and manage query definitions

  • logs_resource_policy module – Create and manage resource policies

  • rds_db_cluster_parameter_group module – Creates and manages an Amazon Aurora DB cluster or Multi-AZ DB cluster

  • rds_db_instance module – Creates and manages resource creates an Amazon DB instance

  • rds_db_proxy module – Create and manage DB proxies

  • rds_db_proxy_endpoint module – Creates and manages a DB proxy endpoint

  • rds_db_subnet_group module – Creates and manages a database subnet group

  • rds_global_cluster module – Creates and manages an Amazon Aurora global database spread across multiple AWS Regions

  • rds_option_group module – Creates and manages an option group

  • redshift_cluster module – Create and manage clusters

  • redshift_endpoint_access module – Creates and manages Redshift-managed VPC endpoint

  • redshift_endpoint_authorization module – Describes an endpoint authorization for authorizing Redshift-managed VPC endpoint access to a cluster across AWS accounts.

  • redshift_event_subscription module – Create and manage Amazon Redshift event notification subscriptions

  • redshift_scheduled_action module – Creates and manages a scheduled action

  • route53_dnssec module – Is used to enable DNSSEC signing in a hosted zone

  • route53_key_signing_key module – Creates a new key-signing key (KSK) in a hosted zone

  • s3_access_point module – Create and manage Amazon S3 access points to use to access S3 buckets

  • s3_bucket module – Create and manage S3 buckets

  • s3_multi_region_access_point module – Create and manage Amazon S3 Multi-Region Access Points

  • s3_multi_region_access_point_policy module – Manage Amazon S3 access policies

  • s3objectlambda_access_point module – Create and manage Object Lambda Access Points used to access S3 buckets

  • s3objectlambda_access_point_policy module – Specifies the Object Lambda Access Point resource policy document

  • ssm_document module – Creates and manages a Systems Manager (SSM) document

  • ssm_resource_data_sync module – Creates and manages a resource data sync

  • wafv2_ip_set module – Creates and manages and IP set

  • wafv2_logging_configuration module – Creates and manages an association between logging destinations and a web ACL resource

  • wafv2_regex_pattern_set module – Creates and manages an AWS WAFv2 Regex Pattern Set

  • wafv2_web_acl_association module – Creates and manages a web ACL association
